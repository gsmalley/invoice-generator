## Bug #1: View Navigation Doesn't Work (CRITICAL)

**Severity:** Critical  
**Status:** Confirmed - Root Cause Found  

### Description
When clicking any navigation button in the sidebar (New Invoice, Clients, Settings), the button shows as active/highlighted but the main content area never changes. The app stays on the "Create New Invoice" form regardless of which nav button is clicked.

### Steps to Reproduce
1. Open http://localhost:5177/
2. Click "Clients" in the sidebar
3. Observe: Button highlights (shows `[active]`) but content stays on invoice form
4. Click "Settings" in the sidebar  
5. Observe: Button highlights but content stays on invoice form

### Root Cause (FOUND)
**Location:** `src/main.ts`, lines 717-730

The view switching event listener code is incorrectly placed **inside the `showUpgradeModal()` function** instead of being in `setupEventListeners()`.

```typescript
// This is WRONG - inside showUpgradeModal() function (line 682)
function showUpgradeModal(isLimitReached: boolean = false) {
  // ... modal code ...
  
  // View switching - ONLY runs when upgrade modal is shown!
  {
    const appEl = document.getElementById('app')!
    appEl.addEventListener('click', (e) => {
      const target = e.target as HTMLElement
      const viewButton = target.closest('[data-view]') as HTMLElement
      
      if (viewButton) {
        const view = viewButton.dataset.view as ViewType
        if (view) {
          currentView = view
          render()
          setupEventListeners()
        }
      }
    })
  }
}
```

The view switching click handler is only attached when the upgrade modal is displayed. On initial page load, there's no event listener to handle view switching, so clicking nav buttons does nothing (even though the sidebar re-renders with the active state because `render()` is called from somewhere else).

### Fix Required
Move the view switching event listener from `showUpgradeModal()` to `setupEventListeners()` function.

## Bug #2: Subscription Status API Fails

**Severity:** Medium  
**Status:** Confirmed - Root Cause Found  

### Description
Console shows error: `Failed to load subscription status: SyntaxError: Unexpected token 'i', "import Str"... is not valid JSON`

### Steps to Reproduce
1. Open http://localhost:5177/
2. Check browser console (F12)
3. Observe: Error appears immediately on page load

### Root Cause (FOUND)
**Location:** `src/subscription.ts`, lines 97-104

The app tries to fetch subscription status from `/api/status`:

```typescript
const url = customerId 
  ? `/api/status?customerId=${encodeURIComponent(customerId)}`
  : '/api/status'

const response = await fetch(url)
if (response.ok) {
  const data = await response.json()  // <-- This fails!
  // ...
}
```

**Problem:** 
1. There is no backend server running (Vite dev server only serves static files)
2. `vite.config.ts` has no API proxy configured
3. When Vite receives a request for `/api/status`, it doesn't know what to do, so it returns the TypeScript source file `subscription.ts` (which starts with `import` statements including Stripe)
4. The code tries to parse this as JSON, which fails

### Fix Required
Either:
1. Set up a backend server to handle `/api/status` requests
2. Configure Vite proxy in `vite.config.ts` to forward `/api/*` requests to a backend
3. Or skip the API call entirely in development mode (use localStorage fallback immediately)
